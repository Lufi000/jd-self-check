# 配置说明

使用本 skill 前，请将 SKILL.md 中出现的以下路径与命名改为你本地的实际配置。

## 需配置的路径

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **自检文档路径** | 存放「工作需求-匹配自检」的 Markdown 文件路径 | `工作需求-匹配自检.md` |
| **简历模板路径** | 用于生成定制简历的基础简历（.md）路径 | `你的简历模板.md` |
| **看板路径** | 岗位投递看板文件路径（Obsidian Kanban 或普通 Markdown） | `岗位投递看板.md` |
| **工作考虑文档约定** | 工作考虑文档的目录与文件名格式 | `{公司}/{公司} {岗位} 工作考虑.md` |

在 SKILL.md 的「数据来源」与「配置说明」小节中，将上述示例路径替换为你工作区内的真实路径即可。定制简历的输出文件名格式也可按需约定（如 `{岗位名称}_简历.md` 或 `{公司}/{岗位名称}_简历.md`）。

## 自备简历要求

为便于 skill 生成「定制简历」与导出 PDF，建议自备简历满足：

- **格式**：**Markdown（.md）**，且包含以下部分（无则定制时可能无法自动调整）：
  - **头部**：姓名、联系方式、求职意向、期望城市、期望薪资
  - **个人优势**（列表或段落）
  - **工作经历**
  - **项目经历**
  - **教育经历**
- 各节标题需清晰可识别，以便按 JD 调整个人优势顺序与措辞、更新头部求职意向与城市等。

**其他格式（如 PDF/Word）**：

- **建立画像（场景零）**：可接受。可粘贴简历文字或提供文件路径；若 Agent 无法解析 PDF，会请你粘贴关键内容（经历、技能等）。
- **定制简历与导出 PDF**：依赖可编辑的 .md 模板。若仅有 PDF/Word，请先另存或复制为一份 .md 作为简历模板后再使用定制简历功能；否则需自行按 JD 修改简历后导出。

## 模板文件

仓库 `templates/` 下提供：

- `工作需求-匹配自检-模板.md`：自检文档第二节～第九节结构，可复制到工作区后填写或由场景零访谈生成内容。
- `岗位投递看板-示例.md`：看板格式示例（四列：待投递、已投递、面试中、结果出）。
- `工作考虑-示例.md`：工作考虑文档的章节结构示例。
- `访谈问题清单.md`：场景零访谈时可参考的提问示例。

上述模板与示例中公司名、岗位名等均为占位（如「某公司」「示例岗位」），不含真实信息。
