# 使用流程说明

## 一、首次使用：建立求职画像

在未具备「工作需求-匹配自检」文档或希望更新画像时：

1. **触发**：对 Agent 说「帮我建立求职画像」「初始化求职自检」，或**直接发简历**，或说「我想找工作」「准备找工作」。若你直接说「对这份 JD 做自检」但尚未有自检文档，Agent 会先引导你完成本步。
2. **提供简历**：粘贴简历全文或给出简历文件路径；若为 PDF/Word，可粘贴关键内容（经历、技能、求职意向等）。
3. **访谈**：Agent 会按自检文档结构（MBTI、经验能力、价值观、特长、技能概览、硬性约束、匹配自检清单）提问，你可逐项回答或跳过可选项。
4. **确认与生成**：问完后 Agent 会呈现求职画像摘要，**请你确认**是否按此写入「工作需求-匹配自检」；确认后再写入，之后即可进行 JD 自检。

## 二、新 JD 自检

1. 对 Agent 说「对这份 JD 做自检」或「这是新的意向岗位」，并**粘贴 JD 全文**或**给出工作考虑文档路径**。
2. Agent 会按自检清单逐条对照，若有无法判断的项会先列出「待你补充」并提问；你补充后会写回自检文档，再输出完整结论（逐条自检结果、硬性约束、综合匹配度、是否建议投递、沟通时必问项）。
3. 结论输出后，Agent 会追问：是否将结论写入某份工作考虑文档？若值得投，还会追问是否添加到投递看板、是否生成定制简历。
4. 定制简历生成后，确认内容无误可追问导出 PDF；导出成功后会自动生成一段 Boss 直聘打招呼语。

## 三、投递与状态更新

- **投递前**：说「我想投递 XX」可生成该岗位的 Boss 打招呼语（若已有工作考虑文档与画像）。
- **已投递**：说「我已经投递了 XX」「XX 已投递」→ Agent 会将看板中该岗位从「待投递」移至「已投递」。
- **面试邀请**：说「XX 约面试了」「收到 XX 的面试邀请」→ Agent 会将岗位移至「面试中」列，并询问是否需要面试准备；若需要，会输出岗位要求回顾、你的匹配点、可能问题与回答要点。

## 四、建议

- 自检文档、看板、工作考虑文档建议放在同一工作区内，路径在 SKILL 中配置一次即可。
- 若你修改过 SKILL 中的路径，在 `git pull` 更新后若有冲突，请以本地路径为准或从 CONFIG 备忘中恢复。
